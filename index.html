Inventory + Cloud + Async Processing

Smart Retail Cart & Inventory Intelligence Platform

GitHub Repo

Problem Statement

Retail and warehouse environments need real-time visibility into cart movement, sensor activity, threshold breaches, and inventory-linked operational anomalies without overloading a single backend service.

Tech Stack

Java 17, Spring Boot 3.2, AWS Lambda, API Gateway, DynamoDB, Redis, SNS, SQS FIFO, Docker, Kubernetes, JWT, Swagger, JUnit 5.

Architecture

Event data is ingested through an API layer, validated and scored in backend services, then persisted to DynamoDB, cached in Redis for live views, and pushed through SNS/SQS for notifications and decoupled downstream consumers.

Key Features

  • Threshold-based alerting for live sensor or cart events
  • Redis-backed live dashboard endpoints for recent inventory activity
  • Async alert pipeline with queue decoupling and dead-letter handling
  • Container-ready deployment path for scalable backend rollout

Impact / Results

Improved operational visibility for inventory-style event streams, reduced Lambda execution cost by approximately 25% through tuning, and created a backend design that is scalable, cloud-ready, and friendly to future ML-driven anomaly scoring.
